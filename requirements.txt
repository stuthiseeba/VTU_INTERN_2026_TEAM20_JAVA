DATABASE SETUP
----------------------------------------------------------------

Step 1 — Start MySQL service

Step 2 — Create the database:
  CREATE DATABASE recruitment_system;

Step 3 — Tables are created automatically by Hibernate on
         first run (spring.jpa.hibernate.ddl-auto=update)

  Tables created automatically:
    users         Team 1 — email, password, role, OTP, verification status
    drive         Team 2 — company, eligibility criteria, rounds
    student       Team 2 — CGPA, branch, skills, graduation year
    application   Team 2 — stage and status workflow

Step 4 — Fix role column (run ONCE after first launch):

  If you see error: "Data truncated for column 'role' at row 1"
  Run this in MySQL Workbench:

    ALTER TABLE users MODIFY COLUMN role VARCHAR(20) NOT NULL;

  Why: MySQL created the column before COORDINATOR and TPO were
  added to the Role enum. Hibernate's ddl-auto=update does not
  alter existing column types automatically.

FRONTEND — FEATURES ADDED DURING INTEGRATION
----------------------------------------------------------------

1. React Router (react-router-dom ^6.30.0)
   - URL-based navigation instead of state-based page switching
   - Protected routes — redirect to /login if not authenticated
   - Login state persists across page refreshes (localStorage)

2. Dark / Light Theme Toggle
   - Floating button (bottom-right corner) on every page
   - Light: warm grey (#F4F0EC background, white cards)
   - Dark:  charcoal grey (#1A1A1A background, #242424 cards)
   - Theme preference saved to localStorage

3. Signup OTP Flow
   - Step 1: Fill form (fullName, email, password, role)
   - Step 2: Enter 6-digit OTP sent to email
   - OTP valid for 5 minutes — Resend OTP available
   - If email not configured: OTP printed to Eclipse console
